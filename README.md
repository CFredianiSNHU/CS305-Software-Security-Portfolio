# CS 305 Software Security Portfolio
CS 305 software security portfolio featuring the Artemis Financial Vulnerability Assessment Report and a reflection on secure software development practices.

## CS 305 Portfolio Reflection
## Portfolio Artifact

For my portfolio artifact, I selected the Artemis Financial Vulnerability Assessment Report. Artemis Financial is a financial services company that wanted to improve the security of its software application. The goal of the project was to identify security vulnerabilities and recommend ways to better protect the application and the customer information it stores.

## Reflection

One thing I did well during this project was learning how to identify software security vulnerabilities and understand why secure coding is important. Before this course, I knew security was important but I did not understand how many different areas of software development it affects. This project showed me that security should be part of the development process from the beginning instead of being added at the end. Secure coding helps protect customer information and reduces security risks.

The vulnerability assessment was one of the most valuable parts of the project. At first, understanding the different security risks was challenging because many of the concepts were new to me. As I worked through the project, I became more comfortable identifying vulnerabilities and understanding how they could affect an application. This experience helped me better understand how developers review software and decide what security improvements should be made.

I increased layers of security by reviewing vulnerabilities and recommending secure coding practices. I also learned that protecting an application involves more than writing secure code. It also includes protecting sensitive data, using secure communication, and reviewing software dependencies. In the future, I would continue using vulnerability assessments, code reviews, and security testing to identify problems before software is released.

After making my recommendations, I reviewed the application to make sure the changes would improve security without affecting how the software worked. This helped me understand that security improvements should support the application instead of creating new problems.

Throughout this course, I learned about several tools and security practices that I will use in future projects. Learning about vulnerability assessments, encryption, certificates, and dependency analysis helped me better understand software security. I also learned that testing and reviewing code are important parts of building secure applications.

If I were showing this project to a future employer, I would choose the Artemis Financial Vulnerability Assessment Report. It shows that I can review software for security problems, recommend improvements, and explain technical information in a way that others can understand. I believe this project represents the knowledge and skills I developed throughout this course.
